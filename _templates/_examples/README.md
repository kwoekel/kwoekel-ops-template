# _examples — Opt-in domain blueprints

These are **specialized** templates for specific kinds of work. They are *not* the
default project shape — the default is `_templates/project-skeleton/` (generic,
best-practice, fits any project).

Pull one of these in only when a project genuinely matches its domain.

| Template | Use when the project is… |
|---|---|
| `lead-gen-framework-template.md` | A content-driven lead generation consultancy (service tiers, channel strategy, DM/network playbook) |
| `four-phase-roadmap-template.md` | A lead-gen build moving through foundation → production → first client → scale |

For anything else, start from the generic skeleton.
