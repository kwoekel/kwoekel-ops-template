## Overview

Four phases that move [PROJECT_NAME] from initial setup to a running, scalable operation.

- **Phase 1** establishes the foundation — partnership formalized, internal [PRIMARY_CHANNEL] presence live using [FRAMEWORK_NAME].
- **Phase 2** upgrades production quality by embedding execution standards into every client deliverable.
- **Phase 3** onboards the first paying client end-to-end.
- **Phase 4** formalizes operations so a second client can be onboarded quickly and the business can scale without chaos.

## Phase Details

### Phase 1: Foundation
**Goal**: Partnership is confirmed in writing, [EXECUTION_LEAD]'s [PRIMARY_CHANNEL] presence is active using [FRAMEWORK_NAME] as internal lead gen, and the pipeline gate criteria are defined.

**Depends on**: Nothing (first phase)

**Requirements**: `[REQ-partnership-formalization]`, `[REQ-pipeline-gate]`

**Success Criteria** (what must be TRUE):
1. [EXECUTION_LEAD] and [STRATEGY_PARTNER] have written alignment on referral mechanics, revenue split, approval chain, and termination terms — confirmed before any client work begins
2. [EXECUTION_LEAD]'s [PRIMARY_CHANNEL] profile is optimized as a sales page and the core value prop is published
3. [EXECUTION_LEAD] is posting 3x/week on [PRIMARY_CHANNEL] using the authority/relatability/conversion sequence to attract consulting clients
4. A pipeline gate document exists: specific criteria for the pivot decision, and [STRATEGY_PARTNER] has confirmed how many campaigns are currently in flight
5. [STRATEGY_PARTNER] has been briefed on the service tier offering and provided feedback — tiers are ready to pitch

### Phase 2: Production Standards
**Goal**: Execution standards from [FRAMEWORK_NAME] are built into reusable templates and workflow steps that every client delivery will use — so the first client gets a production-quality experience from day one.

**Depends on**: Phase 1

**Requirements**: `[REQ-value-prop-translation]`, `[REQ-content-calendar-sequencing]`, `[REQ-performance-reporting]`, `[REQ-profile-setup-sprint]`

**Success Criteria** (what must be TRUE):
1. A reusable onboarding checklist exists with a mandatory "value prop translation" step — one [PRIMARY_CHANNEL]-ready sentence that every future post is evaluated against
2. A content calendar template exists implementing the authority/relatability/conversion rotation with hook structures
3. A Profile Setup Sprint checklist exists (headline, banner, about, featured section, skills audit) with a clear decision on tier placement
4. A reporting workflow exists that generates and sends the monthly performance report — tested against at least one dummy client

**Plans**: TBD

### Phase 3: First Client
**Goal**: The first paying client ([TIER_ENTRY] or [TIER_MID] tier) is live — brand intake complete, profile sprint delivered, content calendar running, approval workflow functional, and first month of posts approved and published.

**Depends on**: Phase 2

**Requirements**: `[REQ-client-brand-intake]`, `[REQ-approval-workflow]`

**Success Criteria** (what must be TRUE):
1. Client has completed brand intake and voice call; their AI system prompt in [AUTOMATION_TOOL] is populated from the framework output
2. Client's [PRIMARY_CHANNEL] profile has been through the Profile Setup Sprint — updated and live before first post
3. Client can view, approve, and request revisions on post drafts in [CONTENT_MGMT_TOOL] with no additional platform login required
4. First 12 posts (one month of content) are approved and scheduled in [PUBLISHING_TOOL] following the authority/relatability/conversion sequence
5. Client receives first monthly performance summary via email; metrics are populated and report is readable

### Phase 4: Scale
**Goal**: The operation can onboard a second client quickly, tooling decisions are finalized, the [ADDON_NAME] add-on is designed and ready to offer, and the top tier is positioned as the compounding tier.

**Depends on**: Phase 3

**Requirements**: `[REQ-addon-module]`, `[REQ-top-tier-reposition]`, `[REQ-onboarding-sop]`

**Success Criteria** (what must be TRUE):
1. An onboarding SOP exists covering every step (stack spin-up, [SECRETS_TOOL] population, [AUTOMATION_TOOL] import, [CONTENT_MGMT_TOOL] base, [PUBLISHING_TOOL] connection, AI prompt setup) — second client onboarded in ≤45 minutes following it
2. Tooling stack is confirmed and documented — tools are in use for first client with no pending decisions
3. The [ADDON_NAME] add-on module has a written scope, pricing, and workflow design — ready to pitch to existing clients or new prospects
4. Top tier service description is updated with compounding tier language and advanced creator features included as deliverables
5. Second client is active and first month of content is running without [EXECUTION_LEAD] inventing new process steps
